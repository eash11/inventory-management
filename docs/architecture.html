Vue 3 Frontend FastAPI Backend In-Memory JSON Data No Database
Tech Stack
Frontend — Port 3000
Vue 3 + Vite
Composition API throughout. No UI library — all styling is hand-written CSS (Flexbox + Grid). Custom SVG charts.
vue-router 4 axios Vite 5 i18n (en / ja)
Backend — Port 8001
Python FastAPI
REST API with Pydantic validation. Data loaded from JSON at startup and held in memory. No ORM or database.
FastAPI 0.137 Pydantic v2 Uvicorn CORS open

Request Data Flow
Step 1 — Browser
Filter Selection
User picks from 4 global filters: Time Period, Warehouse, Category, Order Status.

State lives in the useFilters singleton composable, shared across all views via Vue's reactivity system.
→
Step 2 — Vue Component
Watch + Fetch
Each view watches the filter refs. On change, calls api.js with getCurrentFilters(). Loading state shown while request is in flight.
→
Step 3 — api.js
HTTP Request
Centralized API client builds URLSearchParams from filters, skipping any set to 'all'. Uses axios to call http://localhost:8001/api/...
→
Step 4 — FastAPI
Filter In-Memory Data
apply_filters() and filter_by_month() scan Python lists. Quarters like Q1-2025 expand to 3 month strings.
→
Step 5 — Response
Validate + Render
FastAPI validates response through Pydantic models → JSON. Vue stores raw data in ref([]), derived values in computed(). DOM updates reactively.

Filtering Logic / server/main.py
Two utility functions applied sequentially. Original data lists are never mutated.

filtered = [item for item in items

# filter_by_month — direct or quarter
if month.startswith('Q'):
  months = QUARTER_MAP[month]
  # ['2025-01','2025-02','2025-03']
Filter behavior:
• Skip filter entirely if value is 'all'
• Warehouse: exact string match
• Category: case-insensitive match
• Status: case-insensitive match
• Month: '2025-01' substring match on order_date
• Quarter: expands to 3 month strings via QUARTER_MAP

All endpoints are GET. Base URL: http://localhost:8001/api

Key Design Decisions
No Database — In-Memory JSON
All data lives in Python lists loaded at server startup. Eliminates database setup for demos. Trade-off: writes (purchase orders) don't survive a restart, and the dataset must fit in RAM.
Singleton Composables for Shared State
useFilters and useI18n hold their refs at module scope, so every component that imports them shares the same reactive state — no Vuex/Pinia needed at this scale.
All Charts are Custom SVG
No charting library (no Chart.js, no ECharts). Donut charts, bar charts, and trend lines are built with raw SVG and CSS. Reduces bundle size; makes styling fully controllable.
Filter Logic Lives in the Backend
Filtering is done in Python on the server, not in the browser. Each filter change triggers a fresh API call. This mirrors how a real database-backed API would work and keeps the frontend data flow simple.
Computed Properties for Derived Data
Raw data is stored in ref([]). Everything derived — top products, category totals, low stock alerts, order health metrics — is a computed() property that recalculates automatically when the underlying ref changes.
Internationalisation with Currency Switching
Selecting Japanese also switches the currency display from USD to JPY. Translation keys cover product names, warehouse names, categories, and customer names — not just UI labels.
